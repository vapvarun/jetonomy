A3 — GET /bookmarks callback cleanup
====================================
Generated: 2026-04-29

Background
----------
A1 flagged GET /bookmarks as ⚠️ LOGIN-ONLY:

  Permission callback: '__return_true'
  Handler:             list_items() — calls require_auth()

The callback didn't reflect the handler's actual gate, so the manifest's
`auth` field had to read "login_required_handler" — a special-cased
documentation-only value. Per the work brief this was tagged as a
1-line cosmetic cleanup; we applied it.

Before
------
includes/api/class-bookmarks-controller.php, line 28–32:

  [
      'methods'             => \WP_REST_Server::READABLE,
      'callback'            => [ $this, 'list_items' ],
      'permission_callback' => '__return_true',
      'args'                => $this->get_collection_params(),
  ],

After
-----
includes/api/class-bookmarks-controller.php (with explanatory comment):

  [
      'methods'             => \WP_REST_Server::READABLE,
      'callback'            => [ $this, 'list_items' ],
      // A3 cleanup: handler enforces login via require_auth(); the
      // callback now mirrors that so the manifest's `auth` field is
      // accurate ("login_required" instead of "login_required_handler").
      'permission_callback' => function () { return is_user_logged_in(); },
      'args'                => $this->get_collection_params(),
  ],

Behaviour change
----------------
- Anonymous request: previously hit the handler (which still rejected
  via require_auth() with rest_forbidden). Now rejected at the callback
  level — same HTTP status (401), but slightly faster path (no model
  hydration before the rejection).
- Logged-in user: identical behaviour.

Verified via `bin/access-matrix-check.sh`:

  PASS GET /bookmarks [anon]       expected=401,403 got=401
  PASS GET /bookmarks [subscriber] expected=200      got=200
  PASS GET /bookmarks [admin]      expected=200      got=200

Manifest follow-up
------------------
A future manifest refresh (likely in A6 or A7 when the audit pipeline
re-runs) will pick up the change: GET /bookmarks → auth=login_required.
Not blocking this phase.
