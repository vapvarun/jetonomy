# Jetonomy v1.2.0 — Feature Plan

## Completed in v1.1.0

- Configurable Community Title (H1 on home page)
- Adapter-specific rule type dropdowns (Tutor Course, LearnDash Course, etc.)
- Searchable autocomplete for membership levels (scales to 1000+ courses)
- Human-readable labels in access rules table
- Sync Members button for existing enrollments
- Remove membership on deactivation (all adapters)
- wpForo multi-board import support
- Asgaros importer column mapping fix (verified against actual plugin)
- PHPStan baseline updates

## Planned for v1.2.0

### 1. Private Topics (per-topic visibility)
- Users can mark individual topics as private in a public space
- Only the author + space moderators/admins can see private topics
- Use case: sensitive questions in open course forums, support requests
- Implementation: add `is_private` tinyint column to `jt_posts`, filter in queries, badge in UI

### 2. Topic Prefixes
- Colored labels like "Bug", "Suggestion", "Solved", "Announcement" before topic title
- Admin configures prefixes per space
- Leverage existing tags system with `is_prefix` flag — no new table

### 3. Similar Topics (Duplicate Detection)
- As user types topic title, show "Similar topics" below the title field
- Default: search current space (FULLTEXT on content_plain)
- Toggle: "Search all spaces" expands community-wide
- Top 5 matches with title, space name, reply count

### 4. Quote Replies
- Click "Quote" on any reply → quoted text auto-inserted into editor
- Styled blockquote with author name and link to original

### 5. Bookmarks (Save for Later)
- Extend existing `jt_subscriptions` with a type flag (notify vs bookmark)
- No new table — separate "Bookmarks" tab on user profile

## Discarded

- **Guest posting** — breaks notifications, invites spam, no accountability
- **Activity timeline page** — backend/admin only for now, expose as block/widget later
