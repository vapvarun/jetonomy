# Jetonomy v1.2.0 — Feature Plan

## Confirmed Features

### 1. Private Topics (per-topic visibility)
- Users can mark individual topics as private in a public space
- Only the author + space moderators/admins can see private topics
- Use case: sensitive questions in open course forums, support requests
- Implementation: add `is_private` tinyint column to `jt_posts`, filter in queries, badge in UI
- Permission check in Permission_Engine + REST API + template views

### 2. Topic Prefixes
- Colored labels like "Bug", "Suggestion", "Solved", "Announcement" shown before topic title in listings
- Admin configures prefixes per space (each space can have its own set)
- User selects prefix when creating a topic (optional or required per space setting)
- Leverage existing tags system with a `is_prefix` flag — no new table needed
- Renders as badge/pill in topic list and single topic view

### 3. Similar Topics (Duplicate Detection)
- As user types topic title on new post form, show "Similar topics" below the title field
- Default: search current space only via FULLTEXT on jt_posts.content_plain
- Toggle: "Search all spaces" expands to community-wide
- Shows top 5 matches with title, space name, reply count
- Clicking a match opens it — user can decide if their question is already answered
- No AI needed — FULLTEXT match on title + content_plain is sufficient
- Fires on debounced input (300ms) after 5+ characters typed

### 4. Quote Replies
- Click "Quote" on any reply → quoted text auto-inserted into reply editor
- Renders as styled blockquote with author name and link to original
- Essential forum UX — every major forum has this
- Implementation: "Quote" button per reply, JS inserts blockquote HTML into editor, CSS styling

### 5. Bookmarks (Save for Later)
- Extend existing `jt_subscriptions` table with a type flag (notify vs bookmark)
- No new table — reuse existing infrastructure
- Separate "Bookmarks" tab on user profile
- Different intent from subscriptions: save to read later, not get notified

## Discarded

- **Guest posting** — breaks notifications, invites spam, no accountability
- **Activity timeline page** — backend/admin only for now, expose as block/widget later
