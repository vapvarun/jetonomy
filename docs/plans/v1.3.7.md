# Jetonomy 1.3.7 — Compose Topic Block + Shortcode

> **For agentic workers:** Implement task-by-task. Each task has a verify step. Do NOT mark a task done until its verify step passes. Browser verification is non-negotiable for any UI task (per project CLAUDE.md).

**Goal:** Ship a `jetonomy/compose-topic` block + `[jetonomy_compose_topic]` shortcode that lets any logged-in user create a topic from any WordPress page — either locked to a fixed space or with a picker of spaces the user can post in.

**Architecture:** Single render callback pattern (same as existing `forum-feed`, `trending`, etc.) — `Blocks::render_compose_topic()` delegates to `Shortcodes::compose_topic()`, which renders a new shared partial `templates/partials/compose-topic-embed.php`. Page-builder friendly: shortcode works anywhere, block lives in the `widgets` category so it appears in the inserter on any page. Asset enqueueing is on-demand (detect block presence + shortcode presence) so no overhead on pages that don't use it.

**Tech Stack:** PHP 8.1+, WordPress 6.7+, Interactivity API (existing `jetonomy` store), server-side rendered blocks (api_version 3), PHPUnit, Playwright MCP for browser verification.

**Scope boundary:** Free plugin only. Pro plugin gets a lockstep version bump (`1.3.7`) with no code changes — its existing extensions (polls, AI, etc.) automatically participate via the same hooks the `/new/` page already fires, because we reuse the same form fields.

  Add the method:

  ```php
  /**
   * [jetonomy_compose_topic mode="picker|fixed" space_id="" types="topic,question,idea"]
   */
  public static function compose_topic( $atts ): string {
      $atts = shortcode_atts(
          array(
              'mode'     => 'picker',
              'space_id' => 0,
              'types'    => 'topic,question,idea',
          ),
          $atts,
          'jetonomy_compose_topic'
      );

      $mode      = in_array( $atts['mode'], [ 'fixed', 'picker' ], true ) ? $atts['mode'] : 'picker';
      $space_id  = absint( $atts['space_id'] );
      $types     = array_values( array_filter( array_map( 'trim', explode( ',', (string) $atts['types'] ) ) ) );
      $space     = null;
      $postable  = array();

      if ( $space_id ) {
          $space = \Jetonomy\Models\Space::find( $space_id );
      }

      if ( 'picker' === $mode || ! $space ) {
          $postable = self::_postable_spaces_for_current_user();
      }

      // Ensure assets get enqueued when rendered via shortcode on non-community pages.
      wp_enqueue_style( 'jetonomy-blocks' );
      wp_enqueue_script( 'jetonomy-compose-topic' );

      ob_start();
      \Jetonomy\Template_Loader::partial(
          'compose-topic-embed',
          array(
              'mode'     => $mode,
              'space_id' => $space_id,
              'space'    => $space,
              'postable' => $postable,
              'types'    => $types ?: array( 'topic', 'question', 'idea' ),
          )
      );
      return (string) ob_get_clean();
  }

  /**
   * Spaces the current user can create a post in. Empty array for logged-out.
   *
   * @return \Jetonomy\Models\Space[]
   */
  private static function _postable_spaces_for_current_user(): array {
      $user_id = get_current_user_id();
      if ( ! $user_id ) {
          return array();
      }
      $all = \Jetonomy\Models\Space::all_visible_to( $user_id );
      return array_values( array_filter( $all, static function ( $s ) use ( $user_id ) {
          return \Jetonomy\Permissions\Permission_Engine::can( $user_id, 'create_post', (int) $s->id );
      } ) );
  }
  ```

  > **Note:** `Space::all_visible_to()` may not exist yet. If so, use `Space::find_all()` and filter by visibility inline, matching the pattern already used in `Spaces_Controller::get_items()`. Check the model before wiring.

      ctx.error = '';
  },
  composeTopicTitleInput: ( e ) => {

  },

      const ctx = getContext();
      ctx.body = e.target.value;
  },
  *composeTopicSubmit() {
      const ctx = getContext();
      ctx.error = '';

      if ( ! ctx.spaceId ) {
          ctx.error = wp.i18n.__( 'Choose a space first.', 'jetonomy' );
          return;
      }
      if ( ! ctx.title.trim() ) {
          ctx.error = wp.i18n.__( 'Title is required.', 'jetonomy' );
          return;
      }

      ctx.submitting = true;
      try {
          const res = yield fetch( window.jetonomy.rest + 'posts', {
              method: 'POST',
              credentials: 'same-origin',
              headers: {
                  'Content-Type': 'application/json',
                  'X-WP-Nonce': window.jetonomy.nonce,
              },
              body: JSON.stringify( {
                  space_id:  ctx.spaceId,
                  title:     ctx.title,
                  content:   ctx.body,
                  post_type: ctx.postType,
              } ),
          } );

          if ( ! res.ok ) {
              const err = yield res.json().catch( () => ( {} ) );
              ctx.error = err.message || wp.i18n.__( 'Could not create topic.', 'jetonomy' );
              ctx.submitting = false;
              return;
          }

          const created = yield res.json();
          if ( created && created.permalink ) {
              window.location.href = created.permalink;
              return;
          }
          // Fallback: hard reload.
          window.location.reload();
      } catch ( err ) {
          ctx.error = wp.i18n.__( 'Network error — try again.', 'jetonomy' );
          ctx.submitting = false;
      }
  },
