# 1.5.0 — Model Site + Media Kit (and docs reshoot)

> Plan-first record for three linked workstreams. Approved design (2026-06-13):
> full wipe + reseed → **model site first** → extend `Demo_Seeder` → full brand kit;
> logo = **vector rebuild + modernize**. Docs accuracy audit runs in parallel and
> feeds the reshoot list.

## Why these are one pipeline

The **model site** is the single backdrop both the docs **screenshots** (audit
reshoot) and the media-kit **product visuals** are captured from. The media-kit
**brand foundation** (vector logo, BRAND.md, favicons, social/OG banners) is
generated from vector art + HTML templates and needs **no live site**, so it can
run in parallel. Only product/UI shots wait for the clean model site.

```
clean + reseed model site ──┬──> docs screenshot reshoot (per audit)
                            └──> media-kit product/UI shots
brand foundation (logo/BRAND/social/OG) ── parallel, no site needed
```

## Workstream A — Model Community (model site)

### Target (locked)
- **4 categories → 20 spaces** · type mix forum 7 / qa 7 / ideas 2 / feed 4 · one
  `private` + `approval` space (Insiders) for visibility/access screenshots.
- **~220 topics** (~11/space), **~700–900 replies** (long-tail: most 1–6, one
  "hero thread" per space type with 20–40 for the single-topic shot).
- **~20 members**: 1 admin (existing) + 2 moderators + ~17 members, trust levels
  0–5, avatars, bios, staggered joins.
- QA: ~40% accepted answers. Ideas: votes + roadmap statuses. Feed: status posts.
- A few **pending flags** (mod queue), bookmarks/follows, long-tail votes.
- Timestamps spread over **~90 days** (backdate via `created_at` — confirmed
  supported by `Post::create`/`Reply::create`).
- Pro (when active): reactions, 2–3 polls, awarded badges, 1–2 DM threads.

### Category → space map
| Category | Spaces (type) |
|---|---|
| Start Here | Announcements (feed) · Community Guidelines (forum) · Introduce Yourself (forum) · Getting Started (qa) · Events & Office Hours (feed) |
| Product & Engineering | Feature Discussions (forum) · Bug Reports (qa) · Feature Requests (ideas) · API & Integrations (qa) · Release Notes (feed) |
| Community | General Discussion (forum) · Show & Tell (forum) · Off-Topic Lounge (forum) · Wins & Milestones (feed) · Roadmap Ideas (ideas) |
| Help & Support | Help Desk (qa) · Troubleshooting (qa) · Billing & Accounts (forum) · Knowledge Base (qa) · Insiders / Beta (forum, private, approval) |

### Implementation — refactor `includes/admin/ajax/class-demo-seeder.php`
Keep all existing helpers (`create_users`, `batch_vote`, `seed_pro`, badge defs,
accepted-answer + idea-status logic, the `jetonomy_demo_data` manifest cleanup).
Replace the hardcoded literal arrays with three pieces:

1. **Taxonomy definition** — a data array of the 4×20 table above
   (title/slug/type/visibility/join_policy/icon/description/category).
2. **Content library** — hand-curated title + body pools **per space type**
   (forum topics, qa questions + answer bodies, idea bodies, feed statuses,
   reply snippets). No lorem, no runtime AI. Sized so ~220 topics read varied.
3. **Generator** — distributes topics → replies → votes → accepted answers →
   idea statuses → flags across spaces and users with a **fixed seed**
   (`mt_srand(<const>)`) for reproducible builds; backdates `created_at` across
   ~90 days.

### Full-wipe path (destructive, dev-gated)
Add a `--model` / full-reset path that truncates all `jt_*` content
(categories, spaces, posts, replies, votes, flags, members, etc.) + demo users
before seeding. Guard behind explicit flag (and `WP_ENVIRONMENT_TYPE` check).
Driven by `wp jetonomy demo-seed --model` + the admin one-click.

### Verify (per global rule — prove the flow)
- `php -l` + WPCS on the changed file; `wp jetonomy qa-actions` green.
- Run the seed; assert counts (4 / 20 / ~220 / ~800 / ~20) via SQL.
- Browser-verify at desktop + 390px: community home, a forum space, a qa space
  with accepted answer, ideas roadmap, feed, the private Insiders space, a hero
  thread, leaderboard. Dark mode spot-check.

## Workstream B — Media Kit (mirror Learnomy `docs/marketing/brand/`)

Reference: `…/learnomy/docs/marketing/` — Jetonomy already has the marketing
**copy/campaigns/seo/video** in its `marketing/` tree; the **gap is the entire
`brand/` subtree**. Build it under `jetonomy/marketing/brand/` (GitHub-only, no
publishing).

### Logo (vector rebuild + modernize)
Current state: PNG only, tiny (291×80), **no vector source** → the "not crisp"
root cause. Concept (keep, refine): gradient rounded-tile mark with a
**community/members glyph**, split wordmark **Jet** (ink) + **onomy** (accent
gradient), tagline "Community platform for WordPress". Modernize: cleaner glyph,
gradient locked to `--jt-accent #3B82F6` → indigo, tighter wordmark.

Deliverables (mirror Learnomy):
- `brand/logo/*.svg` — mark (gradient), mark-flat, mark-mono, horizontal (light),
  horizontal-dark, wordmark, favicon + `_preview.html`.
- `brand/logo/png/` — export matrix: mark 16→1024, horizontal 1x/2x light/dark/
  white, wordmark 2x + white, favicon.ico, apple-touch (180), PWA (192/512).
- `brand/social/` — avatar-square, x-header, linkedin-page, linkedin-personal,
  facebook-cover, instagram-square, youtube-cover, bluesky-banner — each from an
  HTML/CSS template in `brand/social/src/` (+ `social.css`).
- `brand/BRAND.md` — brand sheet: logo concept + files table, clear space / min
  size, do/don't, **color tokens pulled from the real `--jt-*` palette**, type,
  chart palette.
- OG / featured (1200×630) for store + social.

### Render approach
SVG authored by hand; PNG matrix + social banners rendered from SVG/HTML via the
Playwright MCP (screenshot of `src/*.html` at exact dimensions) — no standalone
scripts. Show the modernized logo to the user before mass-exporting.
