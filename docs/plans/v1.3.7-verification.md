## TikTok short-URL fix (Basecamp 9809230457 / Zoho 39463)

**Customer:** info@luvnft.com (LUV NFT). Bought Jetonomy specifically for TikTok embeds. Reported on 2026-04-06 that pasting TikTok share URLs only renders a plain hyperlink + bare "TikTok – Make Your Day" preview card instead of the player.

**Root cause:** TikTok's mobile Share button hands out `https://www.tiktok.com/t/<code>/` short URLs. WP core's hardcoded TikTok provider regex matches the canonical `tiktok.com/@user/video/<id>` form only, so `_wp_oembed_get_object()->get_provider($shortUrl, ['discover' => false])` returns false → `OEmbed_Provider::supports()` returns false → we never call `get_data()` → fall through to OG parser → only `<title>` tag survives.

**Fix:** `OEmbed_Provider::maybe_resolve_short_url()` follows one cached HEAD request to resolve the short URL into the canonical form before consulting WP's provider list. The same resolver covers `tiktok.com/t/`, `t.co/`, `redd.it/`, `reddit.com/r/<sub>/s/<code>`, `fb.watch/`, `spoti.fi/`. The cache is a 1-day positive transient (`jt_oembed_short_<md5>`), 1-hour negative on failure. `x.com` URLs get a string rewrite to `twitter.com` (no HTTP). Cross-host redirects to unrelated domains are treated as suspicious and discarded.

**Theme bleed fix:** TikTok / Instagram / Twitter use a `<blockquote>` as their SEO container. Theme default blockquote rules (4px left border, italic, tinted background) were leaking through. Added a scoped reset on `.tiktok-embed`, `.instagram-media`, `.twitter-tweet` inside `.jt-embed`, `.jt-link-embed`, `.jt-post-body`, `.jt-reply-body`. Site-wide quote styling untouched.

**Verified:**

| Test | Result |
|------|--------|
| `wp eval` calling `Preview_Service::fetch('https://www.tiktok.com/t/ZTkmBPTjF/')` | provider=`tiktok`, type=`video`, embed_html=791 bytes |
| Browser repro post on `general-discussion/t/tiktok-short-url-repro-1-3-7-release-gate/` | `<blockquote class="tiktok-embed">` rendered, `tiktok.com/embed.js` enqueued, TikTok player iframe hydrated, 0 console errors from us |
| Computed style on the TikTok blockquote after fix | `border-left: 0px none`, `background: transparent`, `padding: 0`, `quotes: none`, `font-style: normal` |
| Other URL forms (x.com, redd.it, t.co) | All resolved correctly via direct provider unit test in WP-CLI |
