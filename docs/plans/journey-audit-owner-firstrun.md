## Top 3 Gaps

1. **Wizard JS hardcodes `'community/'` as the initial URL preview** (`assets/js/setup-wizard.js:18`). If the site was previously configured with a different `base_slug` (e.g. `discuss`), the progress preview shows the wrong URL on first render; it only corrects after the owner types. The "Visit Community" button on step 3 is also set from server-rendered PHP using the pre-save `$base_slug`, so if the owner changes the slug in step 1 and goes straight to step 2 without any input event, the button can link to the old URL. **Severity: major** - this is the most visible moment of the entire first-run flow.

2. **Trust level thresholds in Settings have no "what does this level unlock?" column.** The admin sees numeric inputs (posts, days, reputation) but nothing that explains what advancing from TL0 to TL2 actually grants (e.g. TL2 unlocks `create_spaces`). An owner tuning thresholds has no way to understand the consequence of their changes without reading source code. **Severity: major** - owners misconfigure trust levels because the contract is hidden.

3. **No "is my community ready to launch?" checklist or signal anywhere in the admin.** After the wizard completes, there is no surface that tells the owner whether email is configured, whether a CAPTCHA is enabled, whether demo data has been removed, or whether at least one non-admin member can read/post. The dashboard note about demo data is the only contextual signal. **Severity: major** - owners go live with broken email, sample content still visible, and no spam protection because there is no prompt to check.
