### 1. Topic Move Between Spaces (Card 9720670878)

**What:** Moderators can move a topic from one space to another via the post "..." dropdown menu.

**Backend:**
- REST endpoint `POST /posts/{id}/move` with `target_space_id` — already existed, fully working
- Permission: `move_posts` action in Permission_Engine (moderators + admins)
- Updates `space_id` on the post, adjusts `post_count` on both source and target spaces

**Frontend:**
- "Move" button in post `jt-more-dropdown` (moderator-only)
- `jetonomySpacePicker()` modal helper — fetches spaces from `GET /jetonomy/v1/spaces`, excludes current space, shows `<select>` dropdown
- `movePost` generator action in view.js — calls REST, shows toast, redirects to new URL
- Icon: `assets/icons/move.svg`

**Files modified:**
- `templates/views/single-post.php` — Move button added after Pin
- `assets/js/view.js` — `jetonomySpacePicker()` + `movePost` action

### 2. Topic Merge (Card 9720670890)

**What:** Merge a duplicate topic into another — moves all replies from source to target, trashes source.

**Backend:**
- `Post::merge_into($source_id, $target_id)` in `includes/models/class-post.php`
  - Moves all replies via `UPDATE replies SET post_id = $target WHERE post_id = $source`
  - Recalculates reply_count on target
  - Trashes source post, decrements space post_count
  - Fires `jetonomy_post_merged` action
- REST endpoint `POST /posts/{id}/merge` with `target_post_id`
  - Requires `move_posts` permission in both spaces
  - Returns prepared target post

**Frontend:**
- "Merge" button in post `jt-more-dropdown` (moderator-only)
- `jetonomyPostPicker()` modal helper — debounced search via `GET /search?q=...&type=post`, click-to-select target, excludes current post
- `mergePost` generator action — confirms via `jetonomyConfirm()`, calls REST, redirects to merged post
- Icon: `assets/icons/merge.svg`

**Files modified:**
- `includes/models/class-post.php` — `merge_into()` method
- `includes/api/class-posts-controller.php` — route + `merge_post()` callback
- `templates/views/single-post.php` — Merge button
- `assets/js/view.js` — `jetonomyPostPicker()` + `mergePost` action
- `includes/class-template-loader.php` — i18n keys: mergeTopicTitle, confirmMerge, topicMerged, mergeFailed

### 3. Reply Split to New Topic (Card 9720670906)

**What:** Extract a reply (and its child replies) into a new standalone topic.

**Backend:**
- `Reply::split_to_topic($reply_id, $new_title, $target_space_id)` in `includes/models/class-reply.php`
  - Creates new Post from reply content + provided title
  - Moves child replies (`parent_id = reply_id`) to new post, resets parent_id to 0
  - Trashes original reply, decrements source post reply_count
  - Fires `jetonomy_reply_split` action
- REST endpoint `POST /replies/{id}/split` with `title` (required) + `space_id` (optional)
  - Requires `move_posts` permission in source space (and target if different)
  - Returns new post id, title, slug, space_slug

**Frontend:**
- "Split to Topic" button in reply `jt-more-dropdown` (moderator-only)
- Uses `jetonomyPrompt()` modal to collect new topic title
- `splitReply` generator action — calls REST, shows toast, redirects to new topic
- Icon: `assets/icons/split.svg`

**Files modified:**
- `includes/models/class-reply.php` — `split_to_topic()` method
- `includes/api/class-replies-controller.php` — route + `split_reply()` callback
- `templates/partials/reply-card.php` — Split to Topic button
- `assets/js/view.js` — `splitReply` action
- `includes/class-template-loader.php` — i18n keys: splitReplyTitle, replySplit, splitFailed

### 4. reCAPTCHA / Cloudflare Turnstile Integration (Card 9720671011)

**What:** CAPTCHA protection on new post and reply forms. Supports reCAPTCHA v3 (invisible) and Cloudflare Turnstile. Trusted users (TL2+) and admins are auto-exempt.

**Backend:**
- Adapter pattern: `Captcha_Adapter` interface in `includes/captcha/interface-captcha-adapter.php`
  - `verify($token, $remote_ip): bool`
  - `get_script_url(): string`
  - `get_site_key(): string`
- `Recaptcha_Adapter` — verifies via Google's siteverify endpoint, supports score threshold
- `Turnstile_Adapter` — verifies via Cloudflare's siteverify endpoint
- `Captcha_Manager` — singleton manager, reads settings, instantiates adapter, enqueues scripts, exposes `verify_or_skip()`
- Auto-skip for `manage_options` users and trust level 2+
- Verification added to `Posts_Controller::create_item()` and `Replies_Controller::create_item()`

**Admin:**
- New "Anti-Spam" tab in Settings page
- Fields: Provider (None/reCAPTCHA v3/Turnstile), Site Key, Secret Key, Score Threshold (reCAPTCHA only)
- Settings stored in `jetonomy_settings` array

**Frontend:**
- `window.jetonomyCaptcha` global set via `wp_add_inline_script` with provider name + site key
- CAPTCHA token retrieved in `submitNewPost` and `submitReply` actions before fetch
- For reCAPTCHA: `grecaptcha.execute(siteKey, { action: 'submit' })`
- For Turnstile: reads `cf-turnstile-response` hidden input value

**Files created:**
- `includes/captcha/interface-captcha-adapter.php`
- `includes/captcha/class-recaptcha-adapter.php`
- `includes/captcha/class-turnstile-adapter.php`
- `includes/captcha/class-captcha-manager.php`

**Files modified:**
- `includes/class-autoloader.php` — `Jetonomy\Captcha\` namespace mapping
- `includes/class-jetonomy.php` — `Captcha_Manager::init()` in `load_dependencies()`
- `includes/api/class-posts-controller.php` — CAPTCHA verification in `create_item()`
- `includes/api/class-replies-controller.php` — CAPTCHA verification in `create_item()`
- `includes/admin/class-admin.php` — sanitization for captcha settings
- `includes/admin/views/settings.php` — Anti-Spam tab UI

### 5. User Online Status — Green Dot (Card 9720677083)

**What:** Show a green dot on user avatars when they were active within the last 5 minutes.

**Backend:**
- `UserProfile::jetonomy_update_last_seen($user_id)` — called in `Template_Loader::render()` on every page load
  - Rate-limited via transient `jetonomy_seen_{user_id}` (1 write per minute max)
  - Updates `last_seen_at` column in `jt_user_profiles`
- `UserProfile::is_online($user_id)` — checks `last_seen_at > NOW() - 5 min`
  - Cached via `wp_cache` key `jetonomy_online_{user_id}` for 60 seconds
  - Prevents N+1 queries on reply lists

**Frontend:**
- CSS class `.jt-avatar-wrap.is-online::after` — green dot (10px circle) positioned bottom-right
- Uses `var(--jt-success)` for color, `var(--jt-radius-full)` for border-radius, `var(--jt-bg)` for border
- Applied to: reply card avatars, profile page avatar, sidebar Top Members
- NOT applied to topic list rows (too many queries per page)

**Files modified:**
- `includes/models/class-user-profile.php` — `update_last_seen()` + `is_online()` methods
- `includes/class-template-loader.php` — `update_last_seen()` call in `render()`
- `templates/partials/reply-card.php` — `jt-avatar-wrap is-online` wrapper
- `templates/views/user-profile.php` — `jt-avatar-wrap is-online` wrapper
- `templates/partials/sidebar.php` — `jt-avatar-wrap is-online` wrapper on Top Members
- `assets/css/jetonomy.css` — `.jt-avatar-wrap` + `.jt-avatar-wrap.is-online::after` styles

### 6. Advanced Search Filters (Card 9720677262)

**What:** Add date range, author, tag, and sort filters to the search page.

**Backend:**
- New REST params on `GET /search`: `date_from`, `date_to`, `author_id`, `tag` (slug), `sort` (relevance/newest/votes)
- `search_posts()` rebuilt with dynamic WHERE clause builder
- Tag filter uses 3-table JOIN (posts + post_tags + tags)
- Sort: `votes` → `vote_score DESC`, `newest` → `created_at DESC`, `relevance` → default
- Hook: `jetonomy_search_query_args` filter fires before query (extensibility)
- Hook: `jetonomy_search_filters` action in template (for Pro extensions to add custom filters)

**Frontend:**
- Collapsible `<details>` filter bar below existing filter pills (All/Posts/Spaces/Tags)
- Fields: Date from, Date to, Tag (text input), Sort (select dropdown)
- Auto-opens when any filter is active
- Apply + Clear buttons
- Mobile responsive: stacks vertically at 640px

**Files modified:**
- `includes/api/class-search-controller.php` — new params, dynamic WHERE, tag JOIN, sort
- `templates/views/search.php` — filter param extraction, filter bar UI, filtered query
- `assets/css/jetonomy.css` — `.jt-search-filters` styles

### 7. Draft Posts + Scheduled Publishing (Card 9720677455)

**What:** Save posts as draft or schedule for future publication.

**Database:**
- `published_at datetime DEFAULT NULL` column added to `jt_posts` in base schema
- `JETONOMY_DB_VERSION` stays at `1.0.0` (pre-release, no migrations needed)

**Backend:**
- `Post::create()` — skips counter increments for draft status
- `Post::publish_scheduled($id)` — publishes a scheduled post, increments counters
- `Post::list_drafts_by_user($user_id)` — lists user's drafts with space info
- `Post::get_due_scheduled()` — finds posts due for publishing
- Cron: `jetonomy_publish_scheduled` runs hourly, publishes due posts
- REST: `status` + `published_at` params accepted in `create_item()`
- REST: `GET /posts/drafts` — lists current user's draft posts
- Draft posts skip moderation, activity tracking, subscriptions, and mention notifications

**Frontend:**
- Split-button on new post form: "Post Topic" + dropdown chevron
- Dropdown options: "Publish now" / "Save as draft" / "Schedule..."
- Schedule panel: datetime input + "Your post will be published automatically" helper text
- Submit button text changes to "Schedule" when scheduling
- Draft toast: "Draft saved" on successful draft creation

**Profile:**
- "Drafts" tab on user profile (author-only, not visible to others)
- Shows draft posts with "Draft" or "Scheduled: {date}" badges
- Each draft links to edit page

**Files created:**
- `includes/db/migrations/class-migration_1_2_0.php` (kept as template, not executed)

**Files modified:**
- `includes/db/class-schema.php` — `published_at` column in posts table

- `includes/api/class-posts-controller.php` — draft/schedule params, `/posts/drafts` endpoint
- `includes/class-router.php` — `drafts` in profile tab URL regex
- `templates/views/user-profile.php` — Drafts tab
