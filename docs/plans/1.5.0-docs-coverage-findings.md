## Section coverage summaries

- **getting-started** (9 gaps): Strong, near-complete onboarding path (overview → install → wizard → first community → in-page auth → admin dashboard). A brand-new owner CAN go install-to-live on these docs alone. Gaps are: inconsistent frontmatter/title across 5 of 6 files, a stale shortcode/widget count in the overview, several flows described with NO screenshot (wizard steps, categories/spaces admin, import, invite link, login block), and the doc ordering puts the always-visible admin dashboard last (order 5) behind the version-specific in-page-auth.
- **admin-settings** (11 gaps): Strong coverage of the 6 Settings tabs and the 7 admin-page docs, with each documented option carrying what-it-does/default/where-to-find. But three real Email-tab fields are undocumented, the License tab has no doc, guest_read is described inconsistently across docs 01 and 07, and 5+ admin pages (Activity Log, Revisions, Categories, Content, Integrations, License) ship with zero screenshots.
- **discussions** (11 gaps): Strong, near-complete coverage of the core discussion flow (topics/replies/voting/drafts/composer/moderation/private+prefixes/modals) with mostly self-explanatory step-by-step prose. Main gaps: the compose-topic block/shortcode (an embeddable composer customers can place on any page) is NOT documented in this customer section at all; voting reputation values are presented as fixed when they are admin-configurable; and several feature-rich screens (composer split-button/draft UI, @mention, accepted-answer callout, prefixes-in-listing, modals) have no screenshots — most docs reuse the same 2 images (new-post-form, single-topic-replies).
- **spaces-and-categories** (11 gaps): Strong, well-written coverage of all 4 space types, both visibility/join-policy axes, per-space settings, access rules, invite links, and the two front-end create/edit flows; but the section named "spaces-AND-categories" contains NO category doc (categories live under admin-settings/12), an enforced Hidden-must-be-invite-only rule is undocumented, and the visual Ideas-roadmap + front-end create/edit flows ship with zero screenshots.
- **user-profiles** (8 gaps): All four surfaces (profiles, leaderboard, online-status, my-spaces) are covered with accurate, mostly self-explanatory docs; main gaps are missing screenshots (my-spaces, edit-profile, dedicated online-dot crop), two shortcode-only surfaces with no customer-facing mention in this section, and one inaccurate "What's Next" claim.
- **moderation-and-trust** (10 gaps): Strong, accurate, mostly self-explanatory section: trust levels, flagging, queue, and anti-spam are all covered and the facts match code (trust names/thresholds, 3 ban types + durations, CAPTCHA providers, rate limits, Akismet/AI layering). Main gaps are the undocumented report-a-USER flow, banning being buried inside the queue doc rather than standing on its own, and missing screenshots for every frontend/interactive moderation surface (report dialog, frontend /mod/ queue, ban dialog, Users admin page).
- **search-and-discovery** (5 gaps): Strong: both core surfaces (full-text search + tags) have self-explanatory docs with screenshots and accurate hook coverage; the main gaps are a missing discovery-surface (Trending) presence in this group and two undocumented sidebar tag hooks.
- **notifications** (8 gaps): Strong, self-explanatory coverage of the bell, full notifications page (filters, delete, bulk), and email delivery/templates/unsubscribe; both referenced screenshots exist. But the customer-facing notification-type list is INCOMPLETE (2 real types missing) and email settings are DUPLICATED across two docs whose defaults tables conflict, which will generate support tickets.
- **integrations** (12 gaps): Strong coverage and clarity — all 12 adapters that exist in code (5 free/Pro membership + 5 LMS + WooCommerce + RCP) plus 3 coexistence integrations (BuddyNext, BuddyPress, FluentCommunity) and theme-compat are documented with step-by-step setup and troubleshooting; the gating gap is screenshots (the core Access Rules / course-search / Sync / Auto-Create screens are never shown, and 3 perfect unused images already sit in the folder), plus some developer-speak in the two coexistence docs.
- **migration** (8 gaps): All 3 real importers (bbPress, wpForo, Asgaros) are documented 1:1 with accurate technical detail (batch=500, dry-run honored only for bbPress, single-pass vs batched resume) — strong coverage; main gaps are a single reused screenshot that can't show the rich import UI states, no section landing page, and a few thin-doc spots (where to find CLI path, what the per-source stats/badges mean, re-import duplication behavior).
- **developer-guide** (8 gaps): Near-complete and self-explanatory for a developer audience: all 8 blocks + 8 shortcodes + 4 widgets documented with attributes and block/shortcode parity called out explicitly, REST (free+Pro) / 140+ hooks / 4 adapters / 13+1 CLI roots / Abilities / template overrides / visibility / modal toolkit / FC + BP integrations all covered. The single systemic gap is screenshots: 7 of 8 blocks have NO "how it displays" image (only compose-topic does), so doc 04 fails the owner's "customer can SEE it" bar for the other surfaces.
- **pro-features** (13 gaps): All 15 Pro extensions + getting-started are doc-covered and the prose is genuinely self-explanatory (steps, where-to-find, what-it-does all present). The section's real weakness is visual: 5 of 15 features have ZERO rendered screenshots (only TODO placeholders), 21 screenshot TODOs remain section-wide, and the AI doc - the special-focus feature - is text-strong but has no image at all. Plus stale "12 Pro features" nav copy and a broken Next-chain.
- **why-jetonomy** (8 gaps): 3 positioning pages (vs-bbpress, vs-wpforo, scalability) are marketing-grade, well-structured, factually accurate against code/manifest (trust 0-5, 20 tables, 68 free + 59 Pro = 127 REST, 4 space types, MP/PMPro-free + Woo/LearnDash/RCP-pro adapters all verified), and each carries a screenshot + cross-links. Main gaps: no section landing/overview page, a missing vs-Asgaros comparison (the importer ships but has no positioning page), no "vs FluentCommunity/Discourse" page, and the two comparison pages reuse the same hero screenshot instead of showing the differentiating features they sell.
