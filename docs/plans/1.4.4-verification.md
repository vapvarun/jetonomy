## Walk steps

### 1. Bundled libraries ship in the zip — 9919462772

Customer contract: "I download the zip, activate, the License screen and background jobs work without running composer or npm." Mandatory for any release built from a clean clone.

Steps:
1. `cd jetonomy && bin/build-release.sh --allow-dirty --skip-browser-smoke --output /tmp` (skip the smoke gate; this is a verification build, not a tag)
2. Inspect the zip's contents for both libs
3. `wp eval` on the live site: `as_enqueue_async_action` is defined → AS booted
4. Browse `/wp-admin/admin.php?page=jetonomy-license` and confirm the Manage License button is clickable and the `edd-sl-sdk.js` resource loads (devtools Network tab)

Pass criteria:
- `unzip -l dist/jetonomy-1.4.4.zip | grep edd-sl-sdk.js` returns one match
- `unzip -l dist/jetonomy-1.4.4.zip | grep action-scheduler.php` returns one match
- License page renders, no `404 edd-sl-sdk.js` in devtools

## UX walk at 390px + dark mode

Function verification covers "does it work". The UX walk covers "does it work the way a phone user in dark mode can use it" - a separate, mandatory gate per the responsive + accessibility rules in CLAUDE.md.

| Surface | 390px layout | Dark mode contrast | Tap target | Outcome |
|---|---|---|---|---|
| Edit Profile form | inputs fit viewport, no overflow | input was 2.5:1 (FAIL), fixed to 14.23:1 (PASS) | 42h (system floor 40px) | fixed in walk |
| Single conversation kebab | items visible after open | 14.23:1 | 41h (system floor) | OK |
| Messages compose | recipient + textarea fit, no overflow | 14.23:1 on inputs | 40h on buttons | OK |
| Single post Report button | no horizontal page overflow | n/a (icon only) | 32w x 44h - width below 44, system pattern for icon-only | system pattern |
| Single space header | inline "Edit space" CTA absent on mobile - admin bar hidden by theme at <600px | n/a | n/a | added inline button in walk |

### UX fixes shipped during the walk

1. **Dark mode form field text contrast.** Reign theme's `input[type="text"]` rule was winning the cascade and forcing `rgb(81, 91, 103)` text on a near-black background (2.5:1 contrast, WCAG AA fail). `.jt-input` now anchors `color`, `background`, `caret-color` to design tokens with `!important`, matching the existing pattern the border and box-shadow lines already use. Same fix applied to `.jt-select` and `.jt-input::placeholder`. Re-verified: 14.23:1.
2. **Mobile-visible Edit Space button.** The admin-bar "Edit this space" entry the 1.4.4 release shipped is hidden on phones (most active themes drop the WP admin bar at <600px). `templates/views/space.php` now renders an inline `.jt-btn-sm.jt-btn-ghost` "Edit space" link under the space title, gated by the same `Permission_Engine::is_space_admin` check. Desktop keeps the admin-bar shortcut; every viewport now has a path.

### Noted but not fixed in 1.4.4

- The system-wide `--jt-tap: 40px` token sets the tap-target floor at 40px (Apple HIG / Material recommend 44+). This is an explicit design choice baked into the CSS for years and applies to every `.jt-btn`, `.jt-input`, and `.jt-act` across the plugin. Bumping the floor is a portfolio-wide refactor that belongs in a future UX foundation pass, not in a release stamped 1.4.4.
- Icon-only action buttons (`.jt-act` like Report, Bookmark, Share, Vote) are 32w x 44h - width below 44. Same system pattern; same future-pass scope.
