# Jetonomy 1.4.0 — Browser QA Checklist

**Audience:** anyone with a browser on this site (or any Jetonomy install) walking the v1.4.0 surfaces before tagging.

**Why this exists:** every 1.4.0 fix touches the front end, so code-quality gates (PHPStan / WPCS / PHPUnit) are necessary but not sufficient. This doc is the human / agent walk that turns "tests pass" into "feature works."

**Time budget:** 45 minutes for a full pass at desktop; +20 minutes for 390px mobile + RTL.

---

## 0. Setup

1. Activate `jetonomy` (free) AND `jetonomy-pro` (Pro). Both must be on `1.4.0`.
2. Open the site as admin: `?autologin=1` on any URL.
3. Seed dedicated QA pages — see [Seeding QA pages](#seeding-qa-pages) below. After seeding the dashboard auto-publishes 19 isolated pages (7 shortcodes + 8 blocks + 4 widgets) plus a QA index.
4. Open Chrome DevTools, set viewport to 390 × 844 for the mobile pass.
5. Optional — toggle `<html dir="rtl">` via the language settings or by adding `<html dir="rtl">` via DevTools for the RTL pass.

## 5. Modal toolkit accessibility

Trigger any confirm dialog (e.g. role change at `/community/s/announcements/members/`). DevTools Console:

```js
const o = document.querySelector('.jt-modal-overlay');
console.log({
  role: o.getAttribute('role'),                     // expect: 'dialog'
  ariaModal: o.getAttribute('aria-modal'),          // expect: 'true'
  hasLabelledBy: !!document.getElementById(o.getAttribute('aria-labelledby')),
  hasDescribedBy: !!document.getElementById(o.getAttribute('aria-describedby')),
  bodyOverflow: document.body.style.overflow,       // expect: 'hidden'

```
Then:
- Press `Tab` repeatedly — focus must cycle inside the dialog (never escapes).
- Press `Shift+Tab` from the first focusable — focus jumps to last.
- Press `Esc` — dialog closes; focus returns to the triggering element.
- Click backdrop — dialog closes.

## 9. Cross-cutting checks

- **Console errors.** DevTools Console must be empty on every QA page (warnings allowed but not red errors).
- **Network 4xx / 5xx.** `/wp-json/jetonomy/v1/*` calls must all return 2xx for the happy path; 401/403 only on permission-blocked actions.
- **No browser-native dialogs.** `window.alert` / `window.confirm` / `window.prompt` must NOT fire anywhere. All confirmations use `window.jetonomyConfirm` / `Alert` / `Prompt`.
- **CSS tokens.** No hardcoded hex / px in the rendered stylesheets — every value should resolve through a `--jt-*` custom property.
- **RTL pass.** Add `<html dir="rtl">`; every QA page must mirror correctly (no off-canvas content, no flipped icons that should not flip).
- **Mobile pass at 390px.** No horizontal scrollbar, every shortcode/widget/block fits, every CTA is reachable.

## Seeding QA pages

Two ways:

### Option A — From the browser (no DB-CLI dependency)

Log in as admin, then in the browser console:

```js
// (See bin/seed-qa-pages.php for the full list of slugs / titles / contents.)
const nonce = wpApiSettings.nonce, apiBase = wpApiSettings.root;
const pages = [
  ['jt-qa-sc-recent-posts', 'QA · Shortcode · Recent Posts', '[jetonomy_recent_posts count="10"]'],
  // … see seed-qa-pages.php for the full set
];
for (const [slug, title, content] of pages) {
  const lookup = await fetch(apiBase+'wp/v2/pages?slug='+slug,
    { credentials:'include', headers:{ 'X-WP-Nonce': nonce }}).then(r=>r.json()).catch(()=>[]);
  const body = JSON.stringify({ title, slug, content, status: 'publish' });
  const target = apiBase + 'wp/v2/pages' + (lookup[0] ? '/'+lookup[0].id : '');
  await fetch(target, { method:'POST', credentials:'include',
    headers:{ 'X-WP-Nonce': nonce, 'Content-Type': 'application/json' }, body });
}
```

### Option B — wp-cli (Local "Open Site Shell")

```bash
wp eval-file wp-content/plugins/jetonomy/bin/seed-qa-pages.php
# To clean up:
wp eval-file wp-content/plugins/jetonomy/bin/seed-qa-pages.php cleanup
```

Both options upsert by slug — re-running overwrites content but keeps the page id, so links to the QA pages stay stable across releases.
