# Jetonomy — JS Internationalization Rule (1.4.3+)

## The Rule

**Every user-facing string in JavaScript flows through a server-localized
i18n object. Never inline literals.**

Three localization channels exist; pick the one matching where the JS runs:

| JS context | Reads from | Server-side source |
|---|---|---|
| Inside the Interactivity store (view.js, pro-view.js) | `state.i18n.<key>` | `wp_interactivity_state( 'jetonomy', [ 'i18n' => [ ... ] ] )` in `class-template-loader.php` |
| Classic frontend JS on community pages | `window.jetonomyData.i18n.<key>` | `wp_localize_script( 'jetonomy-data', 'jetonomyData', [ 'i18n' => [ ... ] ] )` in `class-template-loader.php` |
| Header chrome (`header.js`) | `window.jetonomyHeader.i18n.<key>` | `wp_localize_script` in `templates/partials/header.php` |
| Login block (`login-block.js`) | `window.jetonomyLoginBlock.i18n.<key>` | `wp_localize_script` in `class-blocks.php` |
| Modal toolkit (`jetonomy-modals.js`) | `window.jetonomyModalsI18n.<key>` | `wp_localize_script` in `class-template-loader.php` |
| Page-specific scripts (notifications, space-edit, space-members, new-space) | `window.jetonomyData.i18n.<key>` (shared) | Same as classic frontend — every page script depends on `jetonomy-data` |

## Defense-in-depth fallback

Always include a hard-coded English fallback so the JS still works if the
bridge fails (CSP, script combiners, etc.):

```js
button.textContent = state.i18n?.cancel || 'Cancel';
button.textContent = ( window.jetonomyData?.i18n?.closeMenu ) || 'Close menu';
```

## Format strings with placeholders

Use `%1$d` / `%2$d` / `%s` tokens server-side (translator hint in a PHP
comment) and `.replace()` them in JS:

```php
/* translators: 1: trust level, 2: reputation. */
'trustLevelFormat' => __( 'Level %1$d · %2$d rep', 'jetonomy' ),
```

```js
var fmt = D.i18n.trustLevelFormat || 'Level %1$d · %2$d rep';
trustEl.textContent = fmt
    .replace( '%1$d', level )
    .replace( '%2$d', rep );
```

## Enforcement

`@wordpress/no-unlocalized-strings` is configured as **error** (no baseline)
in `.eslintrc.json`. Run `npm run lint:js` before every commit — zero
errors is the gate.

The Pro plugin mirrors this rule against `window.jetonomyProAdmin.i18n` /
the `jetonomy-pro` text domain.
